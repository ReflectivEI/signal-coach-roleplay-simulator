# PLATFORM GUARDRAILS

## 1) SCORING ENGINE AUTHORITY

The deterministic scoring engine is located at:

- `src/components/roleplay/alignmentEngine.jsx`

Rules:

- This file is the **only** location where scoring math may exist.
- No other file may compute Signal Intelligence scores.
- UI components must only display scores.
- Scores may never be generated by an LLM.

## 2) CANONICAL METRIC IDS

Exactly these 8 metrics are allowed:

- `question_quality`
- `listening_responsiveness`
- `making_it_matter`
- `customer_engagement_cues`
- `objection_handling`
- `conversation_control`
- `adaptability`
- `commitment_gaining`

These IDs may never change.

## 3) METRICS VERSION LOCK

`metricsVersion` must remain:

- `SI-v2-locked-2026`

Any change requires a full SOT migration.

## 4) SCHEMA VALIDATION

Metric outputs must pass:

- `src/lib/signal-intelligence-schema.js`

Schema violations must throw errors.

## 5) ROUNDING STANDARD

All scoring must use:

- `roundHalfUp(value, 1)`

`Math.round` and `toFixed` must never be used for scoring.

## 6) BACKEND API CONTRACTS

Public contracts:

- `POST /api/chat`
- `POST /api/roleplay`

Legacy compatibility endpoints must remain functional:

- `/api/llm/invoke`
- `/api/roleplay/sessions`

## 7) PROMPT SAFETY

LLMs must never:

- generate numeric scores
- assign ratings
- compute metrics

LLMs may only produce qualitative coaching text.

## 8) AGGREGATION LAYER

Aggregation logic exists in:

- `src/lib/eqMetricsAggregator.js`

Rules:

- Aggregation is read-only.
- Aggregation cannot modify underlying scores.

## 9) OBSERVABILITY

Diagnostics exist in:

- `src/lib/scoringDiagnostics.js`

Rules:

- Development-only logging.
- Must never affect runtime scoring.

## 10) FUTURE AGENT RULE

Any automated agent modifying this repository must:

1. Read `docs/PLATFORM_GUARDRAILS.md`.
2. Verify compliance before modifying scoring-related code.
3. Refuse modifications that violate these constraints.

## 11) MESSAGE RENDERING RULES

- Chat messages must preserve punctuation.
- Question marks must never be removed.
- Sanitization may only escape HTML.
- Agents modifying chat rendering must verify punctuation integrity.
